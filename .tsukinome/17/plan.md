# Plan — Issue #17: Add formatDuration helper and integrate into cost summary

> Generated by Tsukinome. The technical blueprint for the work, approved before any code.

## Summary

Add a self-contained formatDuration(ms) utility in a small module under src/, following the repo's convention of pure formatter helpers with JSDoc (as seen in pricing.ts's formatUsd). Wire it into renderCostSummary in src/pipeline/cost.ts wherever a duration in ms is rendered. Cover behaviour and edge cases with a dedicated vitest file matching the existing test style.

## Approach

The repo keeps small pure formatting utilities as named exports with a short JSDoc comment (e.g. formatUsd in src/llm/pricing.ts). I'll follow that idiom rather than introduce a new abstraction. Since the helper is a general-purpose formatter used by the cost summary, I'll place it in a new self-contained module src/format/duration.ts (ESM, .js import extensions per repo convention, type: module). The function normalizes input first: non-finite (NaN, ±Infinity) and negative or zero values return "0ms". For finite positive input it rounds to whole ms, then computes tiers: <1000ms → whole ms + 'ms'; <60000ms → seconds via Math.round(ms/1000) + 's'; <3600000ms → minutes + seconds; ≥3600000ms → hours + minutes (seconds dropped). To satisfy R12 (carry normalization) and R13 (always show lower unit at minute/hour scale), I'll derive components from a single rounded total-seconds (for sub-hour) / total-minutes (for hour scale) value so that rounding that pushes to a full unit carries correctly (e.g. 59999ms → 60s rounds to 1m 0s; a value rounding to 60m carries to next hour). Edge decisions: at second scale use Math.round on seconds but if that rounds to 60 carry up into minute scale; simplest robust implementation is to round ms→seconds once, then bucket on the rounded seconds count. I'll then modify renderCostSummary (and its inputs if a duration field exists) to render durations through formatDuration; the retrieved cost.ts snippet shows only cost/token rendering, so I'll inspect the full file and replace any raw-ms duration rendering with formatDuration, importing it via relative path.
