# Spec — Issue #19: Add formatDuration(ms) helper and apply to cost summary

**Type:** feature

> Generated by Tsukinome. This file is the source of truth for the work; edits here drive the build.

## Summary

Add a self-contained formatDuration(ms) utility that converts a millisecond count into a compact, human-readable string according to time-scale-specific formatting rules, and use it in the run cost summary so durations display in readable form instead of raw milliseconds.

## Requirements

Each requirement is tagged with a confidence level (explicit / inferred / assumption / unknown).

- **R1** `[explicit]` A function formatDuration(ms: number): string exists in a small, self-contained utility module.
- **R2** `[explicit]` For values under one second, the output is the whole-millisecond count followed by 'ms' (e.g., 820 → '820ms').
- **R3** `[explicit]` formatDuration(0) returns '0ms'.
- **R4** `[explicit]` For values from one second up to (but not including) one minute, the output is whole seconds followed by 's' (e.g., 5000 → '5s').
- **R5** `[explicit]` For values from one minute up to (but not including) one hour, the output is 'Xm Ys' where X is whole minutes and Y is remaining whole seconds (e.g., 65000 → '1m 5s').
- **R6** `[explicit]` For values of one hour or more, the output is 'Xh Ym' where X is whole hours and Y is remaining whole minutes, dropping seconds (e.g., 3600000 → '1h 0m').
- **R7** `[explicit]` Sub-second values are rounded to whole milliseconds before formatting, and rounding is applied before scale selection so that a value rounding up to exactly 1000ms is formatted at the seconds scale (e.g., 999.6 → '1s').
- **R8** `[explicit]` Negative or non-finite input (NaN, Infinity, -Infinity) returns '0ms'.
- **R9** `[explicit]` The utility module has accompanying unit tests.
- **R10** `[explicit]` The cost summary display uses formatDuration to render durations instead of raw millisecond values.
- **R11** `[inferred]` For seconds- and minutes-scale values, the seconds/minutes components are computed by truncating (flooring) rather than rounding, per the example 65000 → '1m 5s'.
- **R12** `[inferred]` At the seconds scale and above, sub-second milliseconds are discarded and not displayed.

## Acceptance Criteria

### AC1

- **Given** formatDuration is called
- **When** the argument is 0
- **Then** it returns '0ms'

### AC2

- **Given** formatDuration is called
- **When** the argument is 820
- **Then** it returns '820ms'

### AC3

- **Given** formatDuration is called
- **When** the argument is 5000
- **Then** it returns '5s'

### AC4

- **Given** formatDuration is called
- **When** the argument is 65000
- **Then** it returns '1m 5s'

### AC5

- **Given** formatDuration is called
- **When** the argument is 3600000
- **Then** it returns '1h 0m'

### AC6

- **Given** formatDuration is called
- **When** the argument is a negative number such as -5
- **Then** it returns '0ms'

### AC7

- **Given** formatDuration is called
- **When** the argument is NaN or Infinity or -Infinity
- **Then** it returns '0ms'

### AC8

- **Given** formatDuration is called
- **When** the argument is a fractional sub-second value such as 820.6
- **Then** it rounds to whole milliseconds and returns '821ms'

### AC9

- **Given** formatDuration is called
- **When** the argument is a fractional value that rounds up to exactly 1000ms, such as 999.6
- **Then** it crosses into the seconds scale and returns '1s'

### AC10

- **Given** the run cost summary is rendered with a known duration in milliseconds
- **When** the summary is displayed
- **Then** the duration appears as the formatDuration output rather than a raw millisecond number

## Non-Goals

- Changing how durations are stored or measured internally.
- Adding localization or configurable formatting options.
- Applying the helper to display locations other than the cost summary.

## Edge Cases

- Exactly 1000ms (boundary between ms and seconds scale).
- Exactly 60000ms (boundary between seconds and minutes scale).
- Exactly 3600000ms (boundary into hours scale).
- Fractional millisecond input rounding up across a boundary (e.g., 999.6 rounds to 1000ms and is rendered as '1s').
- Very large durations spanning many hours.
- Non-numeric or undefined/null input passed at runtime despite the numeric type.

## Assumptions

- Seconds and minutes components are computed by flooring the respective remainders rather than rounding.
- At seconds scale and above, leftover sub-second milliseconds are dropped from the output.
